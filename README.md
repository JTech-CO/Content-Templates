# AI 글 작성 템플릿 모음

AI에게 붙여넣거나 링크로 전달해 블로그, 쓰레드, 트위터 글 작성을 돕는 Markdown 템플릿 모음입니다.  
개발용 프로젝트가 아니라, 누구나 원하는 템플릿을 골라 AI에게 전달한 뒤 자신의 주제와 자료를 채워 글을 만들 수 있도록 구성했습니다.

---

## 이런 분께 적합합니다

- 블로그 글을 더 빠르고 일정한 품질로 작성하고 싶은 분
- 체험단, 맛집, 여행, 제품 리뷰 글의 구조를 매번 새로 고민하고 싶지 않은 분
- 하나의 내용을 블로그, 쓰레드, 트위터용으로 재가공하고 싶은 분
- AI에게 더 명확한 글쓰기 지시를 주고 싶은 분
- 글 작성 후 키워드, 해쉬태그, 과장 표현, CTA 등을 점검하고 싶은 분

---

## 사용 방법

1. 원하는 플랫폼 폴더를 엽니다.
   - `블로그 템플릿`
   - `쓰레드 템플릿`
   - `트위터 템플릿`

2. 작성하려는 글의 목적에 맞는 템플릿을 선택합니다.
   - 범용
   - 아티클
   - 체험단
   - 식당
   - 여행
   - 내돈내산 제품 리뷰

3. 템플릿 내용을 AI에게 붙여넣습니다.

4. 템플릿 안의 `[ ]` 영역과 `//// 이 부분 작성 ////` 영역에 자신의 정보를 채웁니다.

5. 필요하면 `부가템플릿`과 `해쉬태그 관리.md`를 함께 사용합니다.

---

## AI에게 요청하는 예시

```text
아래 템플릿을 기준으로 네이버 블로그 글을 작성해 주세요.
대괄호 안의 정보와 내가 추가로 입력한 내용을 반영하고,
템플릿의 작성 조건을 모두 지켜 주세요.

[여기에 선택한 템플릿 내용 붙여넣기]
```

플랫폼 변환을 할 때는 다음처럼 요청할 수 있습니다.

```text
아래 블로그 글을 쓰레드용 게시글로 변환해 주세요.
첫 게시글에는 후킹을 넣고, 마지막 게시글에는 CTA를 넣어 주세요.
각 게시글은 독립적으로 읽혀도 이해되게 작성해 주세요.

[원문 붙여넣기]
[플랫폼 변환 템플릿 붙여넣기]
```

## 해쉬태그 관리

[해쉬태그 관리.md](./해쉬태그%20관리.md)는 플랫폼별 해쉬태그 작성 기준을 담고 있습니다.

- 블로그용: 30~40개 검색 유입형 해쉬태그
- 쓰레드용: 5~10개 관심사 확산형 해쉬태그
- 트위터용: 게시글별 0~2개 최소 태그형 해쉬태그

---

## 추천 사용 흐름

1. `공통 작성 브리프`로 글의 재료를 정리합니다.
2. 플랫폼과 목적에 맞는 메인 템플릿을 고릅니다.
3. AI에게 브리프와 템플릿을 함께 전달합니다.
4. 완성된 글을 `최종 검수 체크리스트`로 점검합니다.
5. 필요한 경우 `금지 표현 및 대체 표현 모음`으로 문장을 다듬습니다.
6. 마지막으로 `해쉬태그 관리.md`를 사용해 플랫폼별 해쉬태그를 만듭니다.

---

## 사용 시 주의사항

- AI가 생성한 내용은 발행 전 반드시 사실 확인을 해주세요.
- 가격, 위치, 운영 시간, 정책, 수치처럼 바뀔 수 있는 정보는 최신 내용을 직접 확인하는 것이 좋습니다.
- 체험단, 협찬, 제품 제공, 원고료 제공이 있는 글은 광고 또는 협찬 사실을 명확히 표시해야 합니다.
- 건강, 식품, 뷰티 관련 글에서는 효과를 단정하거나 보장하는 표현을 피하는 것이 좋습니다.
- 템플릿은 글의 구조를 잡기 위한 도구이며, 최종 책임은 발행자에게 있습니다.

## 빠른 선택 가이드

- 정보성 글을 쓰고 싶다면: `범용`
- 뉴스나 이슈를 정리하고 싶다면: `아티클`
- 협찬 또는 제공받은 후기를 쓰고 싶다면: `체험단`
- 맛집 방문 후기를 쓰고 싶다면: `식당`
- 여행 코스와 방문 팁을 정리하고 싶다면: `여행`
- 직접 구매한 제품 후기를 쓰고 싶다면: `내돈내산 제품 리뷰`
