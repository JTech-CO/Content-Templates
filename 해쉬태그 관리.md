# 해쉬태그 관리

## 사용 목적

위에서 작성한 글의 주제, 키워드, 독자, 플랫폼을 바탕으로 해쉬태그를 생성합니다.  
해쉬태그는 사실과 정보 전달을 목적으로 하며, 글의 내용과 직접 관련 있는 단어만 사용합니다.

---

## 공통 작성 조건

- 한국어 해쉬태그를 중심으로 작성합니다.
- 띄어쓰기가 필요한 단어는 붙여 씁니다.
  - 예: `#서울맛집`, `#내돈내산후기`, `#여행코스`
- 특수문자, 과도한 이모지, 의미 없는 반복 태그는 사용하지 않습니다.
- 본문과 관련 없는 인기 태그는 넣지 않습니다.
- 브랜드명, 제품명, 지역명, 카테고리, 상황 키워드를 균형 있게 포함합니다.
- 최종 출력은 각 해쉬태그 사이를 한 칸씩 띄워 작성합니다.

---

## 1. 블로그용 해쉬태그

### 목적

검색 유입과 주제 분류를 돕는 해쉬태그를 작성합니다.  
네이버 블로그나 검색형 콘텐츠에 맞게 구체적인 키워드를 충분히 포함합니다.

### 작성 조건

- 총 **30~40개** 작성합니다.
- 지역명, 브랜드명, 제품명, 주제명, 카테고리, 상황 키워드를 포함합니다.
- 너무 넓은 단어보다 검색 의도가 분명한 조합형 태그를 우선합니다.
- 맛집, 여행, 제품 리뷰는 실제 검색어처럼 자연스러운 태그를 작성합니다.

### 구성 예시

- 핵심 태그: [주제 / 제품명 / 식당명 / 여행지명]
- 카테고리 태그: [맛집 / 여행 / 제품리뷰 / 체험단 / 아티클 등]
- 상황 태그: [가족외식 / 데이트코스 / 선물추천 / 당일치기여행 등]
- 지역 태그: [서울맛집 / 부산여행 / 강남카페 등]
- 세부 태그: [대표메뉴 / 기능 / 장점 / 사용목적 등]

### 출력 형식

```text
#해쉬태그1 #해쉬태그2 #해쉬태그3 ...
```

---

## 2. 쓰레드용 해쉬태그

### 목적

주제 확산과 관심사 연결을 돕는 해쉬태그를 작성합니다.  
Threads에서는 해쉬태그를 과하게 넣기보다, 글의 흐름을 방해하지 않는 수준으로 사용합니다.

### 작성 조건

- 총 **5~10개** 작성합니다.
- 핵심 주제와 관심사 중심으로 압축합니다.
- 너무 긴 조합형 태그보다 읽기 쉬운 태그를 우선합니다.
- 게시글 본문이 자연스럽게 읽히도록 마지막 게시글 하단에 배치하기 좋게 작성합니다.
- CTA와 어울리는 태그를 포함할 수 있습니다.

### 구성 예시

- 주제 태그: [주제명 / 분야명]
- 관심사 태그: [리뷰 / 여행 / 맛집 / 트렌드 / 기록 등]
- 행동 태그: [저장추천 / 후기공유 / 정보공유 등]

### 출력 형식

```text
#해쉬태그1 #해쉬태그2 #해쉬태그3
```

---

## 3. 트위터용 해쉬태그

### 목적

짧은 글의 가독성을 해치지 않으면서 주제를 분류합니다.  
일반 유저 기준 글자 수 제한을 고려해 해쉬태그는 최소화합니다.

### 작성 조건

- 게시글당 **0~2개**만 사용합니다.
- 꼭 필요한 경우에만 사용합니다.
- 각 게시글 250자 제한을 넘지 않도록 태그까지 포함해 계산합니다.
- 핵심 주제 태그 1개와 보조 태그 1개 정도만 권장합니다.
- 불필요한 태그보다 본문 문장을 우선합니다.

### 구성 예시

- 핵심 태그: [주제명 / 제품명 / 지역명]
- 보조 태그: [후기 / 리뷰 / 여행 / 맛집 / 정보공유]

### 출력 형식

```text
게시글 1 추천 태그: #해쉬태그1 #해쉬태그2
답글 1 추천 태그: #해쉬태그1
답글 2 추천 태그: 사용하지 않음
```

---

## 최종 작성 지시

위에서 작성한 글의 내용과 플랫폼을 바탕으로 해쉬태그를 작성합니다.

- 블로그용이면 30~40개를 한 줄로 작성합니다.
- 쓰레드용이면 5~10개를 한 줄로 작성합니다.
- 트위터용이면 게시글별로 0~2개만 추천합니다.
- 글과 직접 관련 없는 태그는 제외합니다.
- 각 해쉬태그는 한 칸씩 띄어 작성합니다.
